You are a senior SaaS product designer, UX strategist, and frontend architect.

I want you to redesign and improve my web app called **SystemForge**, a habit-building app focused on helping users build systems instead of just setting goals.

Core philosophy:
**“Most people set goals. Few build systems.”**
SystemForge helps users turn vague goals into repeatable daily habits using structured, science-backed steps.

Your mission:
Redesign SystemForge to make it:
1. More beautiful and premium
2. Easier to understand for complete beginners
3. More motivating and less overwhelming
4. More useful through better features, workflows, and micro-interactions
5. Clean, modern, calm, and highly usable
6. Mobile-friendly and accessible
7. Consistent with my current brand identity

## PRODUCT CONTEXT

### Site Name
SystemForge

### Purpose
A habit-building web app that helps users create repeatable personal systems from long-term goals.

### Current Core Features
- **System Builder** — guided 7-step wizard:
  1. Identity
  2. Outcome
  3. Trigger
  4. Action
  5. Reward
  6. Fallback Plan
  7. Review

- **Goal Management** — create, edit, organize long-term goals with categories, priorities, and deadlines
- **Daily Check-ins** — log habits as Done / Partial / Missed with mood tracking and difficulty ratings
- **Templates Library** — pre-built habit systems users can adopt in one click
- **Analytics & Streaks** — track completion, streaks, and consistency over time
- **Journaling** — daily and weekly reflections linked to goals and systems
- **Onboarding** — guided intro for new users to create their first goal/system

## IMPORTANT UX IMPROVEMENTS FOR BEGINNERS

Please redesign the experience using these principles:

### A. Replace complex wording with beginner-friendly wording
For example:
- “Identity” → “Who do you want to become?”
- “Outcome” → “What result are you aiming for?”
- “Trigger” → “When will this habit happen?”
- “Action” → “What is the tiny action?”
- “Reward” → “How will you reinforce it?”
- “Fallback Plan” → “What is the easy version on hard days?”
- “Review” → “How will you reflect and improve?”

Keep the original structure, but make the labels more understandable.

### B. Add progressive disclosure
Don’t show too much at once.
- Show one clear primary action per screen
- Use collapsible help sections
- Reveal advanced analytics later
- Keep core workflows lightweight

### C. Add educational UI
Each important page should teach the user something:
- Tooltips
- Short hints
- Example systems
- Sample templates
- “Try this” suggestions
- “Common mistake” warnings
- “Keep it tiny” reminders

### D. Make the app feel supportive, not judgmental
Avoid harsh red/error emotional design for missed habits.
Use compassionate messaging like:
- “Missed yesterday? Let’s reset gently.”
- “Small actions still count.”
- “Consistency matters more than perfection.”

### 5. System Builder `/systems/new` and `/systems/:id/edit`
This is the most important experience.

Redesign as a premium step-by-step wizard with:
- A clear progress sidebar
- One step per screen
- Short explanations
- Real examples
- Helpful prompts
- “Why this matters” section
- “Good example” and “weak example”
- Progress save
- “Back” and “Next” controls
- Final summary preview before saving

For each step, add clearer beginner prompts:

1. Identity
   - Prompt: “Who do you want to become through this system?”
   - Example: “I want to become someone who takes care of their energy.”

2. Outcome
   - Prompt: “What result are you trying to create?”
   - Example: “Have more energy in the morning and feel healthier.”

3. Trigger
   - Prompt: “When and where will this habit happen?”
   - Example: “After I brush my teeth at 7:00 AM.”

4. Action
   - Prompt: “What small action will you actually take?”
   - Example: “Do 5 minutes of stretching.”

5. Reward
   - Prompt: “How will you make the habit satisfying?”
   - Example: “Mark it complete and enjoy coffee afterward.”

6. Fallback Plan
   - Prompt: “What is the minimum version on hard days?”
   - Example: “Stretch for just 1 minute.”

7. Review
   - Prompt: “How will you reflect and improve over time?”
   - Example: “Every Sunday, review consistency and adjust timing.”

Also:
- Add a live preview panel
- Add a “make it smaller” suggestion engine
- Warn users if the habit feels too ambitious
- Recommend templates based on answers

## ADD MORE USEFUL FEATURES

Please suggest and incorporate useful features that genuinely improve habit-building, especially for beginners.

Prioritize practical features over gimmicks.

### High-value feature ideas to include
1. **Today Mode**
   - A focused daily screen showing only what matters today

2. **Smart Onboarding**
   - Ask 3–5 questions and generate the user’s first system automatically

3. **Quick Add**
   - Create a system or goal from anywhere

4. **Weekly Review Flow**
   - Guided summary of wins, misses, mood, patterns, and adjustments

5. **Recovery Mode**
   - If the user misses multiple days, suggest easier fallback versions rather than punish them

6. **System Health Score**
   - A simple score based on consistency, ease, and clarity

7. **Beginner Tips Layer**
   - Optional helper mode with explanations throughout the app

8. **Reminder Scheduling**
   - Custom reminders tied to triggers and time windows

9. **Mini Habit Version**
   - Every system should support a tiny fallback version

10. **Streak Protection / Compassionate Reset**
   - Reframe missed days so users don’t feel like they failed

11. **Reflection-to-Action Loop**
   - Turn journal insights into system adjustments

12. **Pattern Detection**
   - Help users see what conditions improve success

13. **Celebration Moments**
   - Tasteful milestone moments for 3, 7, 14, 30 day consistency

14. **Focus Mode**
   - A distraction-free screen for completing a habit session

15. **Suggested Templates**
   - Based on selected goal category or onboarding answers

16. **Category-based planning**
   - Fitness, learning, work, mental health, relationships, sleep, etc.

17. **Miss Reason Logging**
   - Let users choose reasons like too hard, forgot, no time, low energy

18. **Adaptive Recommendations**
   - If users repeatedly miss a habit, suggest:
     - smaller action
     - better trigger
     - different time
     - new reward

19. **Goal-to-System Mapping**
   - Every goal should show which systems support it

20. **End-of-day recap**
   - A short summary showing what was done, skipped, learned, and what to improve tomorrow
