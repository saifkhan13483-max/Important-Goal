SECTION 0 — CONTEXT: WHAT THIS APP IS
────────────────────────────────────────────────────────────────────────

SystemForge is a React 18 + TypeScript SPA bundled by Vite 7. It uses:
- Firebase Auth (email/password + Google OAuth) — KEEP AS-IS
- Firestore for all user/goal/system/checkin/journal data — KEEP AS-IS
- Firebase Storage for audio files — this will be REPLACED by Cloudinary
- A mini Node.js HTTP server embedded inside vite.config.ts on port
  35001 that handles two concerns:
    1. Workspace CRUD (persisted to server/workspaces_data.json)
    2. A Groq AI proxy at /api/groq-proxy
- Vite proxies /api/workspace and /api/groq-proxy from port 5000 → 35001
- TanStack Query v5 for server state, Zustand for auth state
- Wouter for routing, shadcn/ui + Tailwind CSS for UI

SECTION 1 — WHAT MUST BE MIGRATED (the two targets)
────────────────────────────────────────────────────────────────────────

TARGET A — WORKSPACE BACKEND
  Currently: A standalone Node.js HTTP server inside vite.config.ts
  persists workspace records to server/workspaces_data.json. Every
  request requires a Firebase ID token in the Authorization header,
  which the server verifies via Google's Identity Toolkit REST API.
  Endpoints: create, join (by invite code), leave, remove member,
  regenerate invite code, sync member stats.

  After migration: All workspace data lives in Firestore inside a
  "workspaces" collection. All the above operations become direct
  Firestore SDK calls from the browser (same pattern as goals/systems).
  The server/workspaces_data.json file and all JSON-file logic are
  deleted. The /api/workspace proxy and server handler are removed from
  vite.config.ts. Firestore security rules enforce authorization in
  place of the manual token-verification step.

TARGET B — FILE / MEDIA STORAGE
  Currently: Audio files (the "future-self" recording feature) are
  uploaded to Firebase Storage, and only the resulting URL is saved to
  the user's Firestore document.

  After migration: Audio (and any other binary files — profile images,
  etc.) are uploaded directly from the browser to Cloudinary using an
  unsigned upload preset. The resulting Cloudinary secure_url is saved
  to the user's Firestore document exactly as before. Firebase Storage
  is no longer used.

TARGET C — GROQ AI PROXY (minor cleanup only)
  The /api/groq-proxy endpoint in vite.config.ts MUST STAY. Do not
  remove it. The only change needed is that once the workspace server
  code is gone, the vite.config.ts server no longer needs the
  token-verification logic (since Firestore rules handle workspace auth
  now). If and only if no other code still calls the token-verifier,
  remove that function. Otherwise leave vite.config.ts alone beyond
  deleting the workspace route handlers.

SECTION 4 — FIRESTORE SECURITY RULES TO ADD
────────────────────────────────────────────────────────────────────────

Add these rules to firestore.rules (create the file if it doesn't
exist). Do NOT overwrite existing rules for other collections — append
or merge:

  match /workspaces/{workspaceId} {

    // Any authenticated user can read a workspace document.
    // The invite-code join flow needs to query by inviteCode field,
    // so reads are open to auth'd users.
    allow read: if request.auth != null;

    // Only the workspace owner can create a new workspace document.
    allow create: if request.auth != null
      && request.resource.data.ownerId == request.auth.uid;

    // Members can update (used for stat sync and leave operations).
    // Owners can update anything. Non-members cannot update.
    allow update: if request.auth != null
      && (resource.data.ownerId == request.auth.uid
          || request.auth.uid in resource.data.members.map(m, m.userId));

    // Only the owner can delete a workspace.
    allow delete: if request.auth != null
      && resource.data.ownerId == request.auth.uid;
  }

----- 5.1  createWorkspace -----
export async function createWorkspace(
  userId: string,
  name: string,
  ownerProfile: { displayName: string; email: string }
): Promise<Workspace>

Logic:
  1. Generate a random 8-character alphanumeric invite code.
  2. Build the workspace object with ownerId = userId, role = "owner",
     members array containing just the owner, createdAt = serverTimestamp().
  3. Call addDoc(collection(db, "workspaces"), workspaceData).
  4. Also update the user's document in the "users" collection:
     set workspaceId = the new document ID.
  5. Return the full workspace object with id set to the new doc ID.

----- 5.2  getWorkspaceById -----
export async function getWorkspaceById(
  workspaceId: string
): Promise<Workspace | null>

Logic: getDoc from workspaces/{workspaceId}. Return null if not found.

----- 5.3  getWorkspaceByInviteCode -----
export async function getWorkspaceByInviteCode(
  inviteCode: string
): Promise<Workspace | null>

Logic:
  query(collection(db, "workspaces"), where("inviteCode", "==", inviteCode))
  Return the first result or null.

----- 5.4  joinWorkspace -----
export async function joinWorkspace(
  inviteCode: string,
  userId: string,
  userProfile: { displayName: string; email: string }
): Promise<Workspace>

Logic:
  1. Call getWorkspaceByInviteCode(inviteCode). Throw if not found.
  2. Check user is not already a member. Throw if already member.
  3. arrayUnion a new member object (role: "member", stats zeroed).
  4. updateDoc the workspace.
  5. updateDoc the user's document: set workspaceId = workspace.id.
  6. Return the updated workspace (re-fetch with getWorkspaceById).

----- 5.5  leaveWorkspace -----
export async function leaveWorkspace(
  workspaceId: string,
  userId: string
): Promise<void>

Logic:
  1. getDoc the workspace. Throw if not found.
  2. If userId === ownerId, throw Error("Owner cannot leave — transfer
     ownership or delete the workspace first.").
  3. Filter the members array to remove the user. updateDoc.
  4. updateDoc user's Firestore document: set workspaceId = null.

----- 5.6  removeMember -----
export async function removeMember(
  workspaceId: string,
  requestingUserId: string,
  targetUserId: string
): Promise<void>

Logic:
  1. getDoc the workspace. Throw if not found.
  2. Verify requestingUserId === ownerId. Throw if not.
  3. Filter members array to remove targetUserId. updateDoc.
  4. updateDoc target user's Firestore document: workspaceId = null.

----- 5.7  regenerateInviteCode -----
export async function regenerateInviteCode(
  workspaceId: string,
  requestingUserId: string
): Promise<string>

Logic:
  1. getDoc. Verify requestingUserId === ownerId.
  2. Generate new 8-char code. updateDoc { inviteCode: newCode }.
  3. Return newCode.

----- 5.8  syncMemberStats -----
export async function syncMemberStats(
  workspaceId: string,
  userId: string,
  stats: MemberStats
): Promise<void>

Logic:
  1. getDoc the workspace.
  2. Map over members array, updating the matching userId's stats object.
  3. updateDoc with the new members array.

(MemberStats is already defined in types/schema.ts — use it.)

----- 5.9  subscribeToWorkspace (real-time listener) -----
export function subscribeToWorkspace(
  workspaceId: string,
  callback: (workspace: Workspace | null) => void
): () => void   // returns unsubscribe function

Logic: onSnapshot(doc(db, "workspaces", workspaceId), snap => callback(...))
Return the unsubscribe function directly.

SECTION 7 — CLOUDINARY MIGRATION (replace Firebase Storage)

Step 7.1 — Create client/src/lib/cloudinary.ts

  export const CLOUDINARY_CLOUD_NAME =
    import.meta.env.VITE_CLOUDINARY_CLOUD_NAME as string;

  export const CLOUDINARY_UPLOAD_PRESET =
    import.meta.env.VITE_CLOUDINARY_UPLOAD_PRESET as string;

  export const CLOUDINARY_UPLOAD_URL =
    `https://api.cloudinary.com/v1_1/${CLOUDINARY_CLOUD_NAME}/upload`;

  /**
   * Uploads a File or Blob to Cloudinary using an unsigned upload preset.
   * Returns the secure_url of the uploaded asset.
   */
  export async function uploadToCloudinary(
    file: File | Blob,
    folder?: string
  ): Promise<string> {
    const formData = new FormData();
    formData.append("file", file);
    formData.append("upload_preset", CLOUDINARY_UPLOAD_PRESET);
    if (folder) formData.append("folder", folder);

    const response = await fetch(CLOUDINARY_UPLOAD_URL, {
      method: "POST",
      body: formData,
    });

    if (!response.ok) {
      const error = await response.json();
      throw new Error(
        error.error?.message ?? "Cloudinary upload failed"
      );
    }

    const data = await response.json();
    return data.secure_url as string;
  }
