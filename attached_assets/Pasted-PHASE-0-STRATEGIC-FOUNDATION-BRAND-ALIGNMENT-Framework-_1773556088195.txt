PHASE 0 — STRATEGIC FOUNDATION & BRAND ALIGNMENT
Framework: D.E.E.P (Deep analysis for strategic decisions)

Role: Act as a world-class SaaS product strategist, brand designer, and UX researcher.

Context
You are redesigning SystemForge, a habit + goal-tracking SaaS app. Before touching any pixel or writing any code, you must establish a rock-solid strategic foundation. Everything in subsequent phases depends on the clarity produced here.

Product Definition
SystemForge turns vague goals into clear daily systems. The core insight the product is built on:

"Goals are the destination. Systems are the daily actions that get you there."

Most people set goals like "get fit," "study more," or "be productive" but have no idea what to do on any given day. SystemForge helps beginners break goals into repeatable daily actions, track consistency, reflect, and improve over time.

Target Audience (Deeply Defined)
Design every decision for these users:

Segment	Age Range	Key Pain Point	Emotional Need
Beginners new to habit tracking	16–35	Don't know where to start	Hand-holding, reassurance
Students	16–24	Overwhelmed by study goals	Structure without rigidity
Young professionals	22–35	Want productivity without burnout	Calm clarity
Self-improvement users	18–40	Have tried and failed other tools	Hope, simplicity
Overwhelmed users	Any	Paralyzed by complex productivity apps	Relief, minimalism
Users needing daily guidance	Any	Lack a repeatable routine	Step-by-step direction

The 5 Core UX Problems to Solve
#	Problem	Success Metric
1	Concept is not instantly understandable — A beginner should understand what SystemForge is, how it helps, why "systems" beat vague goals, and what to do first within 5 seconds of landing.	5-second test clarity
2	Overwhelm — Too much information shown at once. Complex actions aren't broken into guided steps.	Progressive disclosure applied to every multi-step flow
3	Beginner confidence is low — Interface feels demanding, not supportive. Lacks reassuring copy, examples, empty states, and onboarding cues.	Every empty state has a helpful CTA; onboarding completion rate target: 80%+
4	UI doesn't feel premium or beautiful — Needs generous whitespace, soft surfaces, subtle borders, layered cards, smooth hierarchy, tasteful gradients, clean icons, elegant data viz.	Passes a "would I screenshot this?" test
5	Information hierarchy is weak — Primary actions don't stand out: Start Free, Create First Goal, Build First System, Complete Today's Check-In.	Each screen has exactly 1 visually dominant CTA

PHASE 2 — APP UX REDESIGN (In-App Experience)
Framework: F.I.R.S.T + P.A.I.R (Iterative project requiring refinement and collaboration)

Role: Act as a world-class product designer, UX strategist, and interaction designer.

Dependency: Use all strategic decisions from Phase 0 and visual direction from Phase 1.

Core App UX Principles
Every screen must follow these rules:

One primary action per screen — The user should always know what to do next.
Plain language only — No jargon. Explain any non-obvious term the first time it appears.
Visual progress everywhere — Progress bars, rings, streaks, completion counts.
Empty states that teach — Every blank screen is an opportunity to guide, not confuse.
Early success — The first session must feel rewarding (first goal set, first check-in done).
Progressive disclosure — Don't show everything at once. Reveal complexity gradually.
Screen-by-Screen Design Specifications
Screen A: Onboarding Personalization Flow
Purpose: Make the user feel welcomed and understood before they see the main app.
Format: Full-screen step-by-step flow (not a modal). One question per screen.
Progress Indicator: Top bar or stepper dots showing current step out of total.
Questions (one per step):
"What do you want to improve?" — Cards/chips: Health, Fitness, Study, Work, Creativity, Mindfulness, Other
"What's your main goal?" — Short text input with examples: "Exercise 3x a week," "Read daily," "Study for exams"
"How much time can you realistically commit each day?" — Segmented options: 5 min, 15 min, 30 min, 60 min+
"When do you usually have free time?" — Chips: Morning, Afternoon, Evening, Varies
"What motivates you most?" — Cards: Streaks, Progress charts, Journaling, Reminders, Accountability
"Would you like daily reminders?" — Toggle with time picker
"How experienced are you with habit tracking?" — 3 cards: Beginner, Intermediate, Advanced (with friendly descriptions)
Visual Style: Large cards, friendly icons, soft brand-purple accents, encouraging micro-copy at the bottom of each step (e.g., "No pressure — you can always change this later.")
Final Step: "You're all set!" celebration screen with a confetti micro-animation and CTA: "Create Your First Goal"
Screen B: Goal Creation
Purpose: Let the user define a goal in under 60 seconds.
Format: Single-screen form (not a wizard — keep it simple).
Fields:
Goal name (text input, placeholder: "e.g., Exercise 3 times a week")
Category (select dropdown: Health, Study, Work, Creative, Wellness, Custom)
Priority (segmented: Low, Medium, High — with color coding)
Deadline (calendar picker, optional, with helper text: "When do you want to reach this goal?")
Helper Text: Below the form: "Don't overthink it — you can always edit your goal later."
CTA: "Save Goal" → immediately prompts "Build a System for This Goal?"
Empty State (if no goals yet): Illustration + "You haven't set a goal yet. Let's start with one." + CTA: "Create Your First Goal"
Screen C: 7-Step System Builder Wizard
Purpose: The flagship feature. Guide the user through creating a daily system for their goal.

Format: Full-screen wizard with a horizontal stepper at the top.

Interaction model: One question per step. Back/Next buttons. Skip option where appropriate.

Steps:

Step	Title	Content	Input Type
1	Choose Your Goal	Select from existing goals or create a new one	Card selection
2	Define Your Action	"What will you do every day/week?" + examples	Text input + suggestion chips
3	Set Frequency	Daily / X times per week / Weekdays only / Custom	Segmented control
4	Pick a Time or Trigger	"When will you do this?" — Morning / Afternoon / Evening / After [trigger]	Cards + optional time picker
5	Set Difficulty	"What's the minimum version?" — e.g., "Even on a bad day, I'll do at least 5 minutes"	Slider or segmented
6	Reminders	Enable/disable, pick time	Toggle + time picker
7	Review & Start	Summary card of everything, edit links, "Looks good!" CTA	Review card
Design Requirements:

Must feel like a conversation, not a form
Each step has a short motivational hint (e.g., Step 5: "Starting small is the secret to staying consistent.")
Smooth transitions between steps (slide or fade)
Review step shows a polished card with all choices, editable inline
Final CTA: "Start My System" → triggers a subtle celebration animation
Screen D: Daily Check-In
Purpose: The most-used screen. Must be frictionless and satisfying.
Format: Scrollable list of today's systems/habits, each as a card.
Per-habit card:
Habit name + goal it belongs to
3 large tap targets: ✅ Done / 🔶 Partial / ❌ Missed
Optional: quick note field (expandable)
Streak count badge
Top of screen:
Date (today)
Completion ring (X of Y habits done)
Greeting: "Good morning, [Name]. Here's your plan for today."
Bottom of screen:
Mood check (5 emoji-style options: Great, Good, Okay, Low, Rough)
"Finish Check-In" button → celebration animation if all done
Completion state: All done → card turns into a calm congratulatory message: "All done for today. You showed up."
Screen E: Dashboard / Home
Purpose: The main hub. Must feel calm, focused, and action-oriented.
Layout: Single-column with priority-ordered widgets:
Today's Systems (top priority) — Quick check-in access. Shows undone habits.
Streak Summary — Compact visual showing active streaks.
Next Milestone — "3 more days until a 2-week streak!"
One Insight — Plain-language: "You're 12% more consistent than last week."
Recent Journal — Last entry preview with "Write today" CTA.
Analytics Snapshot — Mini chart (weekly completion rate).
Empty state (new user): Friendly illustration + "Welcome to SystemForge! Let's set up your first goal." + CTA button.
Design: Soft card-based layout, generous padding, no data overload. A beginner should scan this in 5 seconds and know: "I should do my check-in."
Screen F: Analytics
Purpose: Show progress in a way non-technical users can understand.
Key Visualizations:
Weekly completion rate (simple bar chart)
Streak timeline (horizontal dots or heatmap-lite)
Best day of week (highlight card)
Mood-vs-completion correlation (simple scatter or overlay)
Plain-Language Insights (prioritized over charts):
"You completed 78% of your planned actions this week."

Weekly — Guided prompts: "What was my biggest win?" / "What pattern do I notice?" / "What should I adjust?"

Design: Warm, distraction-free. Minimal chrome. Soft background. Large text area. Auto-save indicator.
Entry List: Below the editor, show past entries as a reverse-chronological list with date and preview snippet.

PHASE 5 — ACCESSIBILITY & USABILITY
Framework: T.A.G (Straightforward technical task with defined parameters)

Role: Act as a senior accessibility engineer and usability specialist.

Dependency: Applies to all outputs from Phases 1–4.

Requirements Checklist
Color & Contrast:

All text meets WCAG AA contrast ratios (4.5:1 for normal text, 3:1 for large text)
Interactive elements have 3:1 contrast against their background
Status is never communicated by color alone — always pair with icon, label, or pattern
Verify all brand colors in both light and dark mode pass contrast
Typography & Readability:

Minimum 16px body text (never smaller for readable content)
Line height ≥ 1.5 for body text
Maximum line length: 65–75 characters for readability
No text embedded in images
Interactive Elements:

All interactive elements are keyboard accessible (Tab, Enter, Escape, Arrow keys)

PHASE 6 — MOTION DESIGN & MICROINTERACTIONS

Motion Principles
Calm: Motion should reduce cognitive load, not add stimulation.
Purposeful: Every animation communicates something (progress, success, transition, feedback).
Fast: Most transitions 150–300ms. Nothing should feel sluggish.
Consistent: Same easing curve throughout (ease-out or cubic-bezier(0.22, 1, 0.36, 1)).
Microinteraction Specifications
